프로젝트 소개

차량 기반으로 수집한 360 로드뷰·초음파·위치 데이터를 CesiumJS 3D GIS 플랫폼에서 조회·관리하고, 이상징후를 지도에서 확인·조치하는 MVP 운영 플랫폼을 구축합니다.

프로젝트 개요

항목내용

핵심 요구사항 분석

  • CesiumJS 기반 GIS 화면 — 산단 3D 배경, 배관·시설물 위치, 차량 주행 경로, 360 촬영 포인트, 이상 이벤트 위치, 위험등급별 색상, 마커 클릭 상세 조회
  • 360 로드뷰 조회 — 지도 포인트 클릭 시 인스타360 기반 360 이미지/영상 조회, 이전/다음 촬영 지점 이동, 촬영일자·위치·미디어 정보 표시 (Cesium 내 팝업 또는 별도 로드뷰 페이지)
  • 데이터 업로드·관리 — 대용량 360 이미지/영상 업로드, 촬영 좌표·일자·설명 등록, 파일 목록 조회, 촬영 포인트와 미디어 연결 관리
  • 이상징후 이벤트 관리 — 5종 유형(배관 누출 의심, 공사/작업구역, 도로 파손·포트홀, 시설물·표지판 이상, 기타) 등록, 위치 선택, 사진/영상/초음파 첨부, 위험등급, 조치상태·담당자 지정, 조치완료 이력
  • 알람·조치관리 — 신규/미조치/고위험/장기 미처리/조치완료 등 상태별 이벤트 목록 및 관리
  • API 서버 및 연동 구조 — Cesium 데이터 조회, 로드뷰 위치·파일 조회, 이벤트 CRUD, 파일 업로드, 향후 AI 분석 결과 연동 API 구조
  • AI 이상징후 탐지(보조) — 자동 판정이 아닌 후보 탐지·알람 보조 수준. 사전학습 객체탐지 모델 기반 후보 표시 + 향후 전용 모델 연동을 위한 표준 API 구조

제안 포인트

1. 공고의 MVP 전략을 그대로 반영한 단계적 설계. "올해는 완성형 AI가 아닌, 사람이 최종 검토하는 운영 플랫폼"이라는 목표에 맞춰, 핵심 운영 기능(GIS·로드뷰·이벤트·알람)을 먼저 완성하고 AI는 후보 탐지 보조 + 표준 연동 API로 안전하게 확장합니다.

2. 견적을 요청하신 기준대로 명확히 구분. ① MVP 가능 범위, ② AI 포함 시 범위, ③ AI 제외 시 범위, ④ 추가 예산 필요 기능을 견적서에서 분리 제시하여 의사결정을 돕습니다.

3. 공간정보 데이터 정합성 우선. 인스타360·초음파 샘플 데이터와 좌표계(EPSG) 기준을 초기 설계 단계에서 검증하여, 촬영 포인트–미디어–이벤트가 지도상에서 정확히 일치하도록 데이터 연동 규격을 먼저 확정합니다.

4. 대용량 미디어 처리 경험. 대용량 이미지/영상 업로드(멀티파트), 스토리지 연동, 첨부 기반 워크플로우(상태·담당자·이력 관리)는 다수의 관리 시스템·헬스케어 운영 플랫폼에서 검증한 패턴을 적용합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 MVP 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

산단 현장 정보를 차량(인스타360 카메라·초음파 카메라)으로 수집하고, 수집된 360 사진/영상·초음파 이미지·위치정보를 CesiumJS 기반 3D GIS 플랫폼과 연동하여 조회·관리합니다. 올해는 완성형 AI 자동판정이 아닌, 현장 촬영 데이터와 이상 후보를 지도에서 확인하고 사람이 최종 검토·조치하는 MVP 운영 플랫폼 구축을 목표로 합니다.

2. 사용자 정의

사용자 유형설명주요 기능
운영 관리자시스템·데이터·사용자 전반 관리데이터 업로드/관리, 사용자·권한 관리, 전체 이벤트 현황
현장 검토자(담당자)이상징후 검토·조치 수행지도 조회, 로드뷰 확인, 이벤트 등록·조치상태 변경·이력 작성
조회 사용자현황 모니터링GIS 관제 화면 조회, 로드뷰 열람, 상태별 이벤트 목록 확인

3. 기능 요구사항

3.1 CesiumJS 기반 GIS 관제 화면

  • [필수] 산단 지도/3D 배경 표시, 배관·시설물 위치 표시
  • [필수] 차량 주행 경로, 360 촬영 포인트, 이상 이벤트 위치 표시
  • [필수] 위험등급별 색상 구분, 마커 클릭 시 상세정보 조회
  • [선택] 레이어 on/off, 기간·유형별 필터

3.2 360 로드뷰 조회

  • [필수] 지도 포인트 클릭 → 360 이미지/영상 조회, 이전/다음 촬영 지점 이동
  • [필수] 촬영일자·위치정보·미디어 정보 표시
  • [선택] Cesium 내 팝업 방식과 별도 로드뷰 페이지 방식 중 협의 결정

3.3 데이터 업로드·관리

  • [필수] 360 이미지/영상 업로드(대용량 멀티파트), 촬영 좌표 등록
  • [필수] 촬영일자·설명 관리, 파일 목록 조회, 촬영 포인트–미디어 연결 관리

3.4 이상징후 이벤트 관리

  • [필수] 5종 유형(배관 누출 의심, 공사/작업구역, 도로 파손·포트홀, 시설물·표지판 이상, 기타) 등록
  • [필수] 위치 선택, 사진/영상/초음파 결과 첨부, 위험등급 입력
  • [필수] 조치상태 변경, 담당자 지정, 조치완료 이력 저장

3.5 알람·조치관리

  • [필수] 신규/미조치/고위험/장기 미처리/조치완료 목록
  • [필수] 이벤트 상태별 조회 및 관리

3.6 API 서버 및 AI 연동 구조

  • [필수] Cesium 데이터 조회, 로드뷰 위치·파일 조회, 이벤트 CRUD, 파일 업로드 API
  • [필수] 향후 AI 분석 결과 연동을 위한 표준 API 구조 (결과 수신·후보 자동 등록 인터페이스)
  • [보조] 사전학습 객체탐지 모델 기반 이상 후보 탐지 보조(PoC) — 담당자 최종 확인 구조
